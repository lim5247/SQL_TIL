## 1-1. BigQuery 기본지식

~~~
✅ 학습 목표 :
* 데이터 관련 기초 지식(OLTP, SQL, Row, Column, 저장 형태 등)을 설명할 수 있다. 
* BigQuery 관련 기초 지식에 대해서 파악할 수 있다. 
~~~

**데이터 저장 형태**  
데이터베이스(저장소) 테이블(저장공간) 등에 저장  

**데이터 저장 장소**  
MySQL, Oracle, PostgreSQL -> OLTP(거래를 하기 위해 사용되는 DB)에 해당됨  
*OLTP: 보류, 중간 상태가 없음, 데이터 추가 변경 빈번, SQL 사용해서 추출가능 그러나 쿼리 속도가 느림  

**SQL(structed query Language)**  
데이터를 관리하기 위해 설계된 특수목적의 프로그래밍 언어  

**테이블에 저장된 데이터의 형태**  
행row: 가로, 원본 데이터  
열column: 각 데이터의 특정 속성 값  

**OLAP와 데이터 웨어하우스의 등장**  
OLAP: OLTP보다 분석의 속도 기능 나음, 분석을 위한 기능 제공  
데이터웨어하우스(DW): 데이터를 한 곳에 모아 저장 ex) 데이터베이스, 웹(크롤링)파일, API  

**BigQuery**  
BigQuery: 구글 클라우드의 데이터웨어하우스, OLAP+DW  
  특징: 난이도-SQL사용해 쉽게 데이터 추출 가능, 속도-빠름  
  Firebase(앱, 웹), Google Analytics4의 데이터 쉽게 추출할 수 있음  
  데이터웨어하우스를 사용하기 위해 서버(컴퓨터)를 띄울 필요 없음, 구글이 인프라 관리  
  사용하는 이유: Firebase, Google Analytics4를 활용할 경우, 운영을 적은 비용으로 진행하기 위해  
  비용(미국): 퀴리비용, 저장비용
