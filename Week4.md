**데이터 타입** 숫자, 문자, 시간, 날짜, 부울(TRUE, FALSE)

**자료타입 변경하기** CAST()
~~~sql
SELECT
  CAST(1 AS STRING)
~~~

~~~sql
SELECT
  SAFE_CAST('카일스쿨' AS INT64)
~~~

## 4-4. 날짜 및 시간 데이터 이해하기(1) (타임존, UTC, Millisecond, TIMESTAMP/DATETIME)

~~~
✅ 학습 목표 :
* 날짜 및 시간 데이터 타입과 UTC의 개념을 설명할 수 있다. 
* DATE, DATETIME, TIMESTAMP 에 대해서 설명할 수 있다.
* 시간함수들의 종류와 시간의 차이를 추출하는 방법을 설명할 수 있다. 
~~~

**날짜 및 시간 데이터 타입**
- DATE: DATE만 표시, 2023-12-31
- DATETIME: DATE+TIME, TIME ZONE 정보 없음, 2023-12-31 14:00:00
- TIME: TIME만 표시, 23:59:59.00

**TIMEZONE**  
* GMT: 영국의 그리니치 천문대를 기준으로 지역에 따른 시간의 파이를 조정하기 위한 시간의 구분석
  * 영국 근처에서 자주 활용
* UTC: universal time coordinated
  * 국제적인 표준 시간
  * 협정 세계시
  * 타임존이 존재 = 특정지역의 표준 시간대
<br>
* TIMESTAMP: 시간 도장, UTC부터 경과한 시간을 나타내는 값
 * TIMEZONE 정보 있음
 * 2023-12-31 14:00:00 UTC

**작은 시간 단위**  
- millisecond(ms)
 - 천분의 1초, 빠른 반응이 필요할 때 사용
 - Millisecond => TIMESTAMP => DATETIME으로 변경해서 사용
- microsecond(µs)
  - 천분의 1밀리초
- 변환하기
